# Dark Cloud 2 — Georama 2.0 (Time Travel Town Building)

## Overview
Georama evolves from "place pieces" to **"Past actions rewrite Future"**. Geostones from dungeons unlock blueprints; materials + Polyn build them; Culture Points + Conditions alter the Future timeline.

---

## Pipeline: Dungeon → Past → Future

```
DUNGEON (Past Era)
    │
    ├─ Defeat enemies → Geostones drop (themed per dungeon)
    │
    ▼
CARPENTERION (Steve's Database)
    │
    ├─ Insert Geostone → Analyze → Unlocks BLUEPRINTS
    │       (House A, River B, Tree C, NPC D, Factory E...)
    │
    ▼
GEORAMA MODE (Past Map)
    │
    ├─ Select Blueprint → Check Material Cost + Polyn Cost
    │
    ├─ Gather MATERIALS:
    │     • Buy (Shops, increasing stock per chapter)
    │     • Dungeon drops (enemy drops, chests)
    │     • Invent (Photography → Idea → Invention)
    │     • Fish/Aquarium byproducts
    │
    ├─ PLACE on Grid:
    │     • Tile-based (like DC1) but larger maps
    │     • Rotation, elevation (piers, bridges, blocks)
    │     • Terrain constraints (Water, Fire Squall zones)
    │
    ├─ Culture Points (CP) granted per piece
    │
    ├─ Meet CONDITIONS (10 per area):
    │     1. Population ≥ X
    │     2. CP ≥ Y
    │     3. Build Specific: Factory, Tree Type, River Length
    │     4. Place NPC in Specific House
    │     5. NPC Adjacency (e.g., "Near Mayor", "Away from Pub")
    │     6. Terrain Features (Hills, Waterfalls, Gardens)
    │     7. Future-Only Conditions (e.g., "Future Factory Active")
    │
    ▼
FUTURE ALTERATION (Immediate on Condition Meet)
    │
    ├─ New dungeon paths open
    ├─ Enemy tables change
    ├─ Future Chests appear (HP/Def items)
    ├─ Shop inventories expand
    ├─ New Geostones available in Future Dungeons
    └─ Story advances
```

## Culture Points (CP) System

### CP Sources
| Source | CP Value | Notes |
|--------|----------|-------|
| Houses | 50–200 | Larger = more CP but higher cost |
| Attachments | 5–20 | Cheap CP density |
| Nature | 10–100 | Rivers scale with length |
| Functional | 100–500 | High CP, high cost |
| **NPC Residents** | **+50 each** | **Best CP/Polyn ratio** |

### CP Strategy
- **Early**: Spam cheap attachments + houses for CP
- **Mid**: Add functional buildings; recruit NPCs
- **Late**: Optimize for 100% Conditions (specific layouts)

## Conditions (10 per Area)

### Condition Types
| Type | Example | Verification |
|------|---------|--------------|
| **Population** | "20 residents" | Auto-tracked |
| **Culture Points** | "CP ≥ 5000" | Auto-tracked |
| **Building** | "Build Windmill" | Auto-tracked |
| **NPC Placement** | "Place Mayor in Stone House" | Talk to NPC → Request |
| **Adjacency** | "Bakery next to River" | Talk to NPC |
| **Terrain** | "River connects North to South" | Visual + Analysis |
| **Future-Dependent** | "Future Factory operational" | Requires Past+Future loop |

### Condition Rewards
| Completion | Reward |
|------------|--------|
| **Each Condition** | Medal, Material, Polyn, Blueprint |
| **All 10 (100%)** | **Future Chest Key** (HP/Def item), Unique Blueprint, Title |

## Future Alteration Mechanics

### What Changes in Future
| Past Action | Future Result |
|-------------|---------------|
| Build Factory | Future: Industrial zone; new enemies; mechanic NPCs |
| Plant Glowing Trees | Future: Magical forest; fairy enemies; rare fish |
| Connect River | Future: Waterways navigable; boat travel; aquatic enemies |
| Recruit Inventor NPC | Future: Advanced shop; invention discounts |
| Build Power Plant | Future: Electric enemies; Ridepod charging stations |

### Future Chests (Critical for Completion)
- **Appear ONLY when Past 100% Conditions met**
- **Contain**: Fruit of Eden (+10 HP), Defense Item (+5–7 Def), Gourd, Pocket
- **Locations**: Fixed per area; revealed on Future map after condition met
- **Total**: ~30 HP Chests, ~25 Def Chests across game

> **Design Note**: Unlike DC1's random Fruit/Gourd drops, DC2 ties **every permanent stat increase to Georama mastery**. Guaranteed, trackable, fair.

## Materials Economy

### Material Tiers
| Tier | Sources | Examples |
|------|---------|----------|
| **Basic** | Shops (Ch. 1), Early Dungeons | Wood, Stone, Cloth, Scrap Metal |
| **Intermediate** | Shops (Ch. 3+), Mid Dungeons | Gems, Gears, Quality Wood, Copper |
| **Advanced** | Shops (Ch. 5+), Late Dungeons, Inventions | Moon Crystal, Sun Element, High-Grade Gears |
| **Special** | Story, Inventions, Finny Frenzy | Atlamillia Shard, Paznos Part, Ultimate Alloy |

### Material Management
- **Warehouse** (each town) — Infinite storage; shared across towns
- **Auto-sort** — Categories: Building, Invention, Fishing, Weapon, Quest
- **Buy Limits** — Shops restock; some materials limited per chapter

## Comparison: DC1 vs DC2 Georama

| Aspect | DC1 | DC2 |
|--------|-----|-----|
| **Input** | Atla Spheres (direct pieces) | Geostones → Blueprints → Materials |
| **Agency** | Place pre-made pieces | Build from components (more combo) |
| **Progression** | Linear (Village 1 → 2 → 3) | **Causal** (Past → Future → Next Past) |
| **Rewards** | Villager requests → Items | **Conditions → Future Chests (Stats) + Medals** |
| **Constraints** | Grid, requests | Grid, CP, Polyn, Materials, Future-Dependent |
| **Visual Result** | Static village | **Past/Future pair** (see change) |
| **Replayability** | Layout optimization | **Multiple valid solutions per condition** |

---

## Design Analysis

### Strengths
- **Causal satisfaction** — "I built this factory → Future has robots" is powerful
- **Multiple solutions** — Conditions rarely have 1 correct layout
- **CP economy** — Meaningful trade-offs (House vs. Attachments vs. Nature)
- **Future Chests as goals** — Clear, guaranteed payoff for completionists
- **Area identity** — Each Georama has unique constraints (Piers, Fireproof, etc.)

### Weaknesses
- **Material grind** — Late game: thousands of Scrap Metal, Gems
- **Polyn inflation** — Millions needed; farming becomes chore
- **Condition opacity** — Some NPC requests vague ("Nice view")
- **Fire Squall (Heim Rada)** — Punitive; burns non-fireproof buildings randomly
- **No automation** — Place each piece manually; no blueprint save/load
